## 可以提交什么

ReviewPrompt 接受结构化资源，而不是未经整理的 Prompt 堆积。一个有用的提交应该说明：

- 支持的审稿任务。
- 面向的用户角色。
- 预期输入和输出。
- 已知限制和安全风险。
- License、来源和贡献者署名。

## 提交路径

- Submit Prompt：使用 GitHub Issue Form 提交 Prompt。
- Submit Workflow：使用 GitHub Issue Form 提交多步骤流程。
- Submit Skill：使用 GitHub Issue Form 提交可安装 Agent Skill。

## 审核标准

- 任务和目标用户清晰。
- 输出格式关注证据。
- 不编造引用或会议政策。
- 包含可用示例或模板。
- License 兼容。
