## What to Submit

ReviewPrompt accepts structured resources, not unreviewed prompt dumps. A useful submission should explain:

- The review task it supports.
- The role it is designed for.
- The expected input and output.
- Known limitations and safety concerns.
- License, source, and contributor attribution.

## Submission Paths

- Submit Prompt: use the GitHub Issue Form for prompts.
- Submit Workflow: use the GitHub Issue Form for multi-step processes.
- Submit Skill: use the GitHub Issue Form for installable agent skills.

## Review Criteria

- Clear task and target user.
- Evidence-aware output format.
- No fabricated citations or venue policies.
- Usable examples or templates.
- Compatible license.
